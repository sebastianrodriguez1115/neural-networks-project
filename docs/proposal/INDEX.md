# Índice de la carpeta proposal/

1. **`proposal.md`** — Documento de entrega principal. Secciones: Abstract, Objetivo y Alcance, Revisión de Literatura, Caso de Estudio, Conjunto de Datos, Enfoque Propuesto, Resultados Esperados, Evaluación, Consideraciones Éticas.

2. **`proposal_description.md`** — Especificación del curso. Define las secciones requeridas y los entregables (reporte, GitHub, video de 2 min). Leer para saber qué es obligatorio.

3. **`sources.md`** — Todas las citas. 17 fuentes citadas [1]–[17] con resúmenes y URLs, lista en formato IEEE, y fuentes adicionales no citadas. [11] (Lugo & Barreto-Hernández) es el artículo de referencia principal.

4. **`conceptos.md`** — Glosario. Términos de biología/RAM, bases de datos (PATRIC, NDARO), representaciones de k-meros, arquitecturas (MLP, RNN, LSTM, GRU, BiRNN, atención), conceptos de entrenamiento (Adam, desbalance de clases, métricas).

5. **`proposal_draft.md`** — Borrador de trabajo. Notas detalladas del enfoque, definición del problema y justificación de arquitectura. Más detallado que `proposal.md`; útil para redactar secciones nuevas.

6. **`proposal_research.md`** — Guía para escribir la propuesta. Orientación sección por sección sobre qué incluir y errores comunes a evitar.
